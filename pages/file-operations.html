File Operations

RPGLE has powerful, built-in file I/O — no imports needed. Unlike Python where you'd use open() or a database library, RPGLE works directly with DB2 files (tables) on IBM i using operation codes like read, write, chain, update, and delete.

Declaring Files with DCL-F

Before using a file, declare it with dcl-f:

RPGLE
// Input only (read)
dcl-f CUSTFILE disk;

// Full CRUD (read, write, update, delete)
dcl-f CUSTFILE disk usage(*input:*output:*update:*delete);

// Keyed access (using a keyed logical file)
dcl-f CUSTFILE disk keyed;

// Output only (write new records)
dcl-f CUSTFILE disk usage(*output);

// Qualified (fields accessed as CUSTFILE.fieldName)
dcl-f CUSTFILE disk keyed qualified;
📝 Python Comparison

In Python you'd do conn = sqlite3.connect('db') and then cursor.execute('SELECT ...'). In RPGLE, dcl-f binds the program directly to a DB2 file at compile time — no connection string needed. The file's fields automatically become available as program variables.

Reading Records

Sequential Read — Read All Records

Python
import sqlite3
conn = sqlite3.connect('db')
cursor = conn.execute('SELECT * FROM customers')
for row in cursor:
    print(row)
RPGLE
dcl-f CUSTFILE disk;

read CUSTFILE;
dow not %eof(CUSTFILE);
  dsply CUSTNM;
  read CUSTFILE;
enddo;

READE — Read with Equal Key

RPGLE
dcl-f ORDERFILE disk keyed;

// Read all orders for customer 1001
setll 1001 ORDERFILE;
reade 1001 ORDERFILE;
dow not %eof(ORDERFILE);
  // Process each order for customer 1001
  dsply ORDNUM;
  reade 1001 ORDERFILE;
enddo;

CHAIN — Direct Key Lookup

chain reads a specific record by key — like a Python dictionary lookup or SELECT WHERE id = ?:

Python
cursor.execute(
    'SELECT * FROM customers WHERE id = ?',
    (1001,)
)
row = cursor.fetchone()
if row:
    print(row['name'])
RPGLE
dcl-f CUSTFILE disk keyed;

chain 1001 CUSTFILE;
if %found(CUSTFILE);
  dsply CUSTNM;
else;
  dsply 'Not found';
endif;

Composite Key Lookup

RPGLE
// File keyed by company + department
dcl-f DEPTFILE disk keyed;

// Lookup with composite key
chain ('ACME' : 'IT') DEPTFILE;
if %found(DEPTFILE);
  dsply DPTNAM;
endif;

Writing Records

RPGLE
dcl-f CUSTFILE disk usage(*output);

// Set field values
CUSTID = 9999;
CUSTNM = 'New Customer';
CUSTAD = '123 New St';
CUSTBL = 0;

// Write the record
write CUSTREC;   // CUSTREC = record format name

Updating Records

RPGLE
dcl-f CUSTFILE disk keyed usage(*input:*update);

// First, read (lock) the record
chain 1001 CUSTFILE;
if %found(CUSTFILE);
  // Modify fields
  CUSTNM = 'Updated Name';
  CUSTBL = CUSTBL + 100.00;
  // Write changes back
  update CUSTREC;
endif;
⚠️ Record Locking

When you chain or read a record for update, RPGLE locks it. Other programs cannot modify it until you update or unlock it. Always ensure you update or unlock promptly to avoid lock contention.

Deleting Records

RPGLE
dcl-f CUSTFILE disk keyed usage(*input:*delete);

// Delete by key
delete 1001 CUSTFILE;
if %found(CUSTFILE);
  dsply 'Record deleted';
else;
  dsply 'Record not found';
endif;

Positioning — SETLL / SETGT

Operation Meaning Use Case
setll Set Lower Limit Position cursor at or before the key
setgt Set Greater Than Position cursor after the key
RPGLE
dcl-f CUSTFILE disk keyed;

// Check if a key exists (without reading)
setll 1001 CUSTFILE;
if %equal(CUSTFILE);
  dsply 'Key 1001 exists';
endif;

// Read backwards from a key
setgt 1001 CUSTFILE;
readp CUSTFILE;   // Read previous record

Dynamic OPEN / CLOSE

RPGLE
// Declare with USROPN — manual open/close
dcl-f CUSTFILE disk keyed usropn;

// Open when needed
if not %open(CUSTFILE);
  open CUSTFILE;
endif;

// ... do work ...

// Close when done
close CUSTFILE;

Complete CRUD Example

CUSTMAINT.RPGLE
**free
ctl-opt dftactgrp(*no) main(main);

dcl-f CUSTFILE disk keyed
        usage(*input:*output:*update:*delete);

dcl-proc main;
  dcl-s count int(10) inz(0);

  // CREATE — Add a new customer
  CUSTID = 5001;
  CUSTNM = 'Alice Johnson';
  CUSTAD = '123 Oak Ave';
  CUSTBL = 1500.00;
  write CUSTREC;

  // READ — Get customer by key
  chain 5001 CUSTFILE;
  if %found(CUSTFILE);
    dsply ('Found: ' + %trim(CUSTNM));
  endif;

  // UPDATE — Modify the record
  chain 5001 CUSTFILE;
  if %found(CUSTFILE);
    CUSTBL += 250.00;
    update CUSTREC;
  endif;

  // LIST — Read all records
  read CUSTFILE;
  dow not %eof(CUSTFILE);
    count += 1;
    dsply %trim(CUSTNM);
    read CUSTFILE;
  enddo;

  dsply ('Total: ' + %char(count));

  // DELETE — Remove the record
  delete 5001 CUSTFILE;

  *inlr = *on;
end-proc;
RPGLE — File Operation Gallery

// 1. Lookup & Process

if %found(CUSTFILE);

read INPFILE;

  read INPFILE;

✅ RPGLE vs SQL

RPGLE's native file I/O is extremely fast for record-level access. For set-based operations (joins, aggregations), you can also embed SQL directly in RPGLE with exec sql — giving you the best of both worlds!
