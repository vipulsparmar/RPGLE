Getting Started with RPGLE Free Format

Welcome to RPGLE — a powerful business-oriented programming language that has been the backbone of IBM i (AS/400) systems for decades. In this chapter, you'll learn what RPGLE is, how it evolved, and write your very first free format program.

What is RPGLE?

RPGLE (Report Program Generator IV — Extended) is a high-level language used primarily on IBM i (AS/400 / iSeries). It excels at:

  • Processing large volumes of transactional data
  • Working with DB2 databases natively
  • Building backend business logic and batch processing
  • Interfacing with modern technologies via APIs
💡 Think of it this way

If Python is the versatile "Swiss army knife," RPGLE is the "precision engine" tuned for enterprise business operations on IBM i.

History & Evolution

Era Version Key Changes
1960s RPG I & II Punch card–based report generator, column-dependent
1970s–80s RPG III System/38, structured programming concepts
1994 RPG IV (RPGLE) Major modernization: longer names, expressions, subprocedures
2001 Free-format calcs Calculations freed from columns via /free and /end-free
2013 Full Free (V7R1) All specs free format. No column rules at all!
2019+ Ongoing New BIFs, data types, open access, ILE improvements

Free Format vs Fixed Format

In fixed format, code must be placed in specific columns. In free format, you write code like any modern language.

Fixed Format (Old Way)

RPG IV — Fixed Format
     H DFTACTGRP(*NO)
     D myName          S             20A   INZ('Hello')
     D myAge           S              3P 0 INZ(25)
     C                   IF        myAge >= 18
     C     myName        DSPLY
     C                   ENDIF
     C                   EVAL      *INLR = *ON
⚠️ Column Dependency

Putting code in wrong columns causes compilation errors. Very error-prone and hard to read.

Free Format (Modern Way)

RPGLE — Full Free Format
**free
ctl-opt dftactgrp(*no);

dcl-s myName char(20) inz('Hello');
dcl-s myAge  packed(3:0) inz(25);

if myAge >= 18;
  dsply (myName + ' (Age: ' + %char(myAge) + ')');
endif;

*inlr = *on;
✅ Advantage

Free format is easier to read, write, and maintain. This entire tutorial uses full free format exclusively.

Your First RPGLE Program Patterns

HELLO_GALLERY.RPGLE
**free
// Pattern 1: Minimal Hello World
dsply 'Hello World!';

// Pattern 2: Hello with Variables
dcl-s msg varchar(50) inz('Welcome!');
dsply msg;

// Pattern 3: Simple Input/Output
dcl-s name char(10);
dsply 'What is your name?' ' ' name;
dsply ('Hello ' + name);

return;

Line-by-Line Explanation

  • **free — Tells the compiler this source is entirely free format.
  • dsply — Displays a message on the screen.
  • return — Ends the program execution.

RPGLE Program Structure

A typical program follows this order:

  1. ctl-opt — Control options (compiler directives)
  2. dcl-f — File declarations
  3. dcl-s / dcl-c / dcl-ds — Global variables, constants, data structures
  4. dcl-pr — Prototype declarations
  5. Main logic — Executable code or main procedure
  6. dcl-proc / end-proc — Sub-procedures

Compilation Process

CL Command — Simple
// Compile a bound RPG program (one step)
CRTBNDRPG PGM(MYLIB/HELLO) SRCFILE(MYLIB/QRPGLESRC) SRCMBR(HELLO)

// Run it
CALL MYLIB/HELLO

Control Options (CTL-OPT)

Option Purpose Common Values
dftactgrp Use default activation group? *no (always for modern code)
actgrp Activation group name *caller, *new
main Linear main procedure Procedure name
option Compiler options *srcstmt, *nodebugio
datfmt Default date format *iso, *usa, *eur
