Procedures & Subprocedures

Procedures in RPGLE are like Python functions — reusable blocks of code that accept parameters and return values. They are the foundation of modular, maintainable RPG programs.

Overview

RPGLE Keyword Purpose Python Equivalent
dcl-proc Define a procedure def function_name():
dcl-pi Procedure interface (parameters) Function parameters
dcl-pr Prototype (forward declaration) N/A (not needed in Python)
return Return a value return
end-proc End procedure Indentation ends

Basic Procedure (No Parameters)

Python
def sayHello():
    print('Hello!')

sayHello()
RPGLE
**free
dcl-proc sayHello;
  dsply 'Hello from RPGLE!';
end-proc;

sayHello();

Subroutines (The Older Way)

Before modern Procedures, RPG used Subroutines. They are simpler but less powerful: they cannot accept parameters, cannot return values, and all variables are global.

RPGLE — Subroutine
**free
begsr calculateTax;
  taxAmount = orderTotal * 0.15;
endsr;

orderTotal = 500;
exsr calculateTax;  // Execute Subroutine
dsply %char(taxAmount);
⚠️ Subroutines vs Procedures
  • Subroutines: Built-in with begsr / exsr. Global scope only. No parameters.
  • Procedures: Defined with dcl-proc. Local scope. Parameters and Return values supported. Preferred for modern code.

Parameters with DCL-PI

dcl-pi (Procedure Interface) defines the parameters a procedure accepts and its return type:

RPGLE
**free
dcl-proc greetUser;
  dcl-pi *n;
    userName char(30) const;
    age      int(10) const;
  end-pi;

  dsply ('Hello ' + %trim(userName) + ' (Age: ' + %char(age) + ')');
end-proc;

greetUser('Alice' : 30);
📝 Parameters are separated by colons

In RPGLE, function arguments are separated by : (colon), not , (comma) like in Python. So greetUser('Alice' : 30) not greetUser('Alice', 30).

Return Values

Specify the return type on the dcl-pi line:

Python
def add(a, b):
    return a + b

result = add(10, 20)
RPGLE
**free
dcl-proc add;
  dcl-pi *n packed(11:2);
    a packed(11:2) const;
    b packed(11:2) const;
  end-pi;
  return a + b;
end-proc;

dsply ('10 + 20 = ' + %char(add(10 : 20)));

The return type (packed(11:2)) is placed right after dcl-pi *n. If the procedure returns nothing (a "void" function), omit the return type.

Prototypes (DCL-PR)

If a procedure is called before it's defined, you need a prototype (forward declaration). This is not needed in Python but required in RPGLE like C's function prototypes.

RPGLE
**free
ctl-opt dftactgrp(*no) main(main);

// Prototype — declare BEFORE calling
dcl-pr calcTax packed(11:2);
  amount packed(11:2) const;
  rate   packed(5:2) const;
end-pr;

dcl-proc main;
  dcl-s tax packed(11:2);
  tax = calcTax(1000 : 18);  // Works because of prototype
  dsply %char(tax);
end-proc;

// Implementation — defined AFTER call
dcl-proc calcTax;
  dcl-pi *n packed(11:2);
    amount packed(11:2) const;
    rate   packed(5:2) const;
  end-pi;
  return amount * rate / 100;
end-proc;

The CONST Keyword

By default, RPGLE passes parameters by reference (the procedure can modify the caller's variable). Use const to pass by read-only reference (like Python's default behavior for immutable types):

RPGLE
// CONST = read-only. Can pass literals and expressions.
dcl-proc double;
  dcl-pi *n int(10);
    value int(10) const;   // Read-only
  end-pi;
  return value * 2;
end-proc;

result = double(25);  // OK - can pass a literal

// Without CONST — must pass a variable
dcl-proc increment;
  dcl-pi *n;
    value int(10);           // By reference — can modify!
  end-pi;
  value += 1;               // Modifies the caller's variable
end-proc;

Local Variables

Variables declared inside a procedure are local — just like in Python functions:

RPGLE
dcl-proc calculateArea;
  dcl-pi *n packed(11:2);
    length packed(7:2) const;
    width  packed(7:2) const;
  end-pi;

  // Local variables — only visible in this procedure
  dcl-s area packed(11:2);

  area = length * width;
  return area;
end-proc;

EXPORT Keyword

By default, procedures are local to the module. Use export to make them available to other programs/modules (like Python's public functions in a module):

RPGLE
// Visible to other modules that bind to this one
dcl-proc getDiscount export;
  dcl-pi *n packed(5:2);
    customerType char(10) const;
  end-pi;

  select;
    when customerType = 'VIP';
      return 20.00;
    when customerType = 'REGULAR';
      return 5.00;
    other;
      return 0;
  endsl;
end-proc;
RPGLE — Procedures Gallery
**free
// 1. Modern Subprocedure
dcl-proc calculateDiscount;
  dcl-pi *n packed(5:2);
    total packed(7:2) const;
  end-pi;
  return total * 0.10;
end-proc;

// 2. Legacy Subroutine
begsr updateStatus;
  status = 'ACTIVE';
  updatedAt = %timestamp();
endsr;

exsr updateStatus;

return;

Complete Example

ORDERUTIL.RPGLE
**free
ctl-opt dftactgrp(*no) main(main);

// Constants
dcl-c TAX_RATE 0.18;

// Prototypes
dcl-pr calcSubtotal packed(11:2);
  price packed(7:2) const;
  qty   int(10) const;
end-pr;

dcl-pr calcTotal packed(11:2);
  subtotal packed(11:2) const;
end-pr;

// Main procedure
dcl-proc main;
  dcl-s sub   packed(11:2);
  dcl-s total packed(11:2);

  sub   = calcSubtotal(29.99 : 3);
  total = calcTotal(sub);

  dsply ('Total: ' + %char(total));
end-proc;

// Subprocedures
dcl-proc calcSubtotal;
  dcl-pi *n packed(11:2);
    price packed(7:2) const;
    qty   int(10) const;
  end-pi;
  return price * qty;
end-proc;

dcl-proc calcTotal;
  dcl-pi *n packed(11:2);
    subtotal packed(11:2) const;
  end-pi;
  return subtotal + (subtotal * TAX_RATE);
end-proc;
