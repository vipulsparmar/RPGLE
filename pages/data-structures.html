Data Structures

Data structures in RPGLE group related fields together — like Python's class, dict, or dataclass. They're declared with dcl-ds and are one of the most powerful features of RPGLE.

Overview

Python
class Customer:
    def __init__(self):
        self.name = ''
        self.age = 0
        self.balance = 0.0

cust = Customer()
cust.name = 'Alice'
RPGLE
**free
dcl-ds customer qualified;
  name    char(30);
  age     int(10);
  balance packed(11:2);
end-ds;

customer.name = 'Alice';
customer.age = 30;

dsply ('Name: ' + customer.name + ' Age: ' + %char(customer.age));

Basic Data Structure

RPGLE
// Simple DS — fields are standalone in the global namespace
dcl-ds address;
  street  char(50);
  city    char(30);
  state   char(2);
  zipcode char(10);
end-ds;

// Access directly (no prefix needed)
street = '123 Main St';
city   = 'Springfield';
state  = 'IL';

Qualified Data Structures Recommended

With qualified, subfields are accessed using dot notation — like Python objects. This is the recommended approach to avoid name conflicts.

RPGLE
**free
dcl-ds employee qualified;
  id      int(10);
  name    char(50);
  salary  packed(9:2);
end-ds;

employee.id   = 1001;
employee.name = 'Bob Johnson';

dsply ('Emp: ' + %trim(employee.name) + ' (' + %char(employee.id) + ')');
✅ Always use QUALIFIED

Without qualified, subfields become global names — which can clash with other variables. Use qualified to get clean, Python-style object.field access.

LIKEDS — Copy a Structure Layout

Create a new variable with the same layout as an existing DS — like creating another instance of a Python class:

RPGLE
// Define a "template" DS
dcl-ds t_product qualified template;
  id    int(10);
  name  char(50);
  price packed(9:2);
  qty   int(10);
end-ds;

// Create instances using LIKEDS
dcl-s product1 likeds(t_product);
dcl-s product2 likeds(t_product);

product1.name  = 'Widget';
product1.price = 29.99;

product2.name  = 'Gadget';
product2.price = 49.99;
📝 TEMPLATE keyword

A template DS doesn't allocate any storage — it's just a blueprint (like a Python class definition before you create instances). Use it with likeds to create actual variables.

Arrays with DIM

Use dim to create arrays — like Python lists but with a fixed size:

RPGLE
// Simple array
dcl-s scores packed(5:2) dim(10);

scores(1) = 95.50;  // 1-based indexing!
scores(2) = 87.25;

// Array of data structures
dcl-ds employees qualified dim(100);
  id     int(10);
  name   char(50);
  salary packed(9:2);
end-ds;

employees(1).id     = 1001;
employees(1).name   = 'Alice';
employees(1).salary = 65000;

employees(2).id     = 1002;
employees(2).name   = 'Bob';
employees(2).salary = 72000;

// Loop through array
for i = 1 to %elem(employees);
  if employees(i).id > 0;
    dsply employees(i).name;
  endif;
endfor;
⚠️ 1-Based Indexing

RPGLE arrays start at index 1, not 0 like Python. scores(1) is the first element, not scores(0).

Nested Data Structures

RPGLE
dcl-ds t_address qualified template;
  street  char(50);
  city    char(30);
  state   char(2);
  zip     char(10);
end-ds;

dcl-ds customer qualified;
  id       int(10);
  name     char(50);
  homeAddr likeds(t_address);    // Nested!
  workAddr likeds(t_address);    // Nested!
end-ds;

// Access nested fields with chained dot notation
customer.name = 'Alice';
customer.homeAddr.street = '123 Oak Ave';
customer.homeAddr.city   = 'Portland';
customer.workAddr.street = '456 Pine St';

Externally Described DS

RPGLE can automatically create a data structure based on a database file's record format — no manual field definitions needed:

RPGLE
// DS based on a database file (all fields imported)
dcl-ds custRec extname('CUSTFILE') qualified;
end-ds;

// Now custRec has all subfields matching CUSTFILE columns
custRec.CUSTNM = 'New Customer';
custRec.CUSTID = 12345;

Clearing / Initializing a DS

RPGLE
// Reset all subfields to defaults
clear customer;

// Initialize with INZ
dcl-ds config qualified inz;
  maxRetries int(10) inz(3);
  timeout    int(10) inz(30);
  debug      ind     inz(*off);
end-ds;
RPGLE — Data Structure Gallery
**free
// 1. Qualified & Initialized
dcl-ds appConfig qualified inz;
  port    int(10) inz(8080);
  active  ind     inz(*on);
end-ds;

// 2. Multi-occurrence / Array DS
dcl-ds orders qualified dim(50);
  id      int(10);
  total   packed(11:2);
end-ds;

// 3. Data Structure Template
dcl-ds t_user qualified template;
  handle  varchar(20);
  email   varchar(100);
end-ds;

return;
